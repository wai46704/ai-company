# Notion「編集部ボード」実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 編集部のタスク・記事案・下書きの状態管理をNotionデータベース「編集部ボード」に一元化し、社長がスマホで「読む→承認」を完結できるようにする。

**Architecture:** Notionに専用ページ＋データベースを作成し、既存のタスクボード.md・記事案リスト.md・下書きの状態をカードとして移行する。Notionの読み書きは編集長（メインのClaude、claude.ai Notion連携＝MCP経由）のみが行い、AI社員は従来通りローカルファイルに成果を置く。朝会手順（asakai.md）とミナト・ハルの定義を更新する。

**Tech Stack:** Notion MCP（claude.ai連携、ToolSearchでロード）、Markdownファイル、Git

**Spec:** `docs/superpowers/specs/2026-07-08-notion-task-board-design.md`

## Global Constraints

- Notionは社外サービス。体験談ヒアリングの生メモ・個人情報はNotionに書かない（ブログ原稿・X投稿・メルマガ原稿は公開前提なので転記可）
- Notionの読み書きは編集長のみ。AI社員（サブエージェント）にはNotion操作をさせない
- git commit対象は `編集部/`・`.claude/`・`docs/` のみ。`src/` は絶対に含めない
- コミットは `git add <個別パス>` で行う（`git add .` 禁止）
- Notion障害時のフォールバック: 朝会は従来ファイルで続行し社長に報告。勝手に代替ファイルを作らない
- ステータスは7種で固定: 提案中 / 作業中 / 社長の確認待ち / 承認済み / 完了 / 却下 / 保留
- 「📥 社長の承認待ち」ビューのフィルタは **ステータス＝「提案中」または「社長の確認待ち」**（記事案の採用判断も社長の判断事項のため両方表示する）
